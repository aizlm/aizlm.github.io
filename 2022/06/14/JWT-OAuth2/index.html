有状态与无状态

详解【1】

  • Web服务的状态指的是请求的状态，而不是资源的状态

    • 基于状态的服务，简单的说就是当前的用户请求状态与某一台服务器进行绑定，无法进行负载均衡等调度；
    • 基于无状态的服务，Server不保存任何请求状态信息，Client的每一个请求都具有User credentials等所需要的全部信息，所以能被任意可用的Server应答。

  • Restful Api 的 stateless？

    • 之前提交的内容不会成为本次提交的“条件”，本次提交无论怎么提交返回都是相同的结果。就web的例子，一般登录后服务器就记录改用户是有权限的，之后都返回有权限的信息，这叫有状态（记住了登录的状态）；每次提交都得提交认证信息，服务器才返回有权限的信息，这叫无状态。

  • （了解）LocalStorage与sessionStorage：

    • Web存储分为两种，分别对应两个JavaScript对象：
      • 本地存储：对应localStorage对象，用于长期保存网站的数据，并且站内任何页面都可以访问该数据。
      • 会话存储：对应sessionStorage对象，用于临时保存针对一个窗口（或标签页）的数据。
