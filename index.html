　　LEA BX, 6[DI]
解释：某数组含20个元素，每个元素占一个字节，序号为0~19。设DI指向数组开头处，如果把序号为6的元素的偏移地址送到BX中
TYPE*+1 = addr + sizeof(TYPE)

tips

OF overflow flag 溢出标志 操作数超出机器能表示的范围表示溢出,溢出时为1.
SF sign Flag 符号标志 记录运算结果的符号,结果负时为1.
ZF zero flag 零标志 运算结果等于0时为1,否则为0.
CF carry flag 进位标志 最高有效位产生进位时为1,否则为0.
AF auxiliary carry flag 辅助进位标志 运算时,第3位向第4位产生进位时为1,否则为0.
PF parity flag 奇偶标志 运算结果操作数位为1的个数为偶数个时为1,否则为0.
DF direcion flag 方向标志 用于串处理.DF=1时,每次操作后使SI和DI减小.DF=0时则增大.
IF interrupt flag 中断标志 IF=1时,允许CPU响应可屏蔽中断,否则关闭中断.
TF trap flag 陷阱标志 用于调试单步操作.

1-NOP（删除跳转）

2-jmp（改变跳转）

先进入目标软件进行相应字符串的搜集 然后od软件中查找字符串(ascii、unicode、智能搜索 用哪种具体情况具体分析吧) 然后就是查找关键字符 找到相应跳转je语句
发现没有跳转成功 是因为没有运行到相应的位置
双击对应十六进制位或f2下断点 f9运行程序 f8单步运行

  • je跳转时 z标志位为0时不实现 为1时跳转实现 (不能通过去改变寄存器和标志位实现破解保存 保存不了 但是可以实现测试的作用)
    找到刚刚设的断点处 然后选中je条件跳转 space输入【jmp 相应跳转成功地址】**(这一步修改汇编代码时 可能遇到je后面跟着X00000000 修改成jmp时需要在X前加上0或者直接删除X 才能修改成功)

  • 未解码状态下的汇编代码：add byte xxxxx 解码状态下的汇编代码：mov edx,engydt xxxxxx (software\engydt是程序进行验证用户名和验证码访问的注册表\项)
    重启后 发现还是需要注册

    OD中API断点设置工具中GetStartupInfoA是初始化断点 在程序初始化之前设下断点 让程序初始化的时候就停下来 (在有壳的程序中使用)
    b 可以看到当前下的所有断点
    使用初始化断点的目的是为了判断程序是否解码 若是解码了 就将初始化断点禁用掉 然后将注册表项的断点激活
    然后f9使程序运行起来 直接断在注册表项位置 f8单步运行 后面在寄存器一栏中可以看到自己输入的假码和真码(可疑的都去试试 验证成功就是真的 其他那些验证不成功的管他是什么)

加密算法

bash64算法(准确的说是编码算法)

特征:
1、将数据按照 3个字节一组的形式进行处理，每三个字节在编码之后被转换为4个字节。
即：如果一个数据有6个字节，可编码后将包含6/3*4=8个字节
2、当数据的长度无法满足3的倍数的情况下，最后的数据需要进行填充操作，即补“=” ，这里“=”是填充字符，不要理解为第65个字符

ROT13

凯撒n=13时的算法
ROT13（回转13位）是一种简易的替换式密码算法

UUE算法(uuencode)

将二进制文件转换为文本文件的过程
算法过程:
从二进制文件中读取 3字节的数据，表示如下（a7 表示 a字节的第 7位）：
a7a6a5a4a3a2a1a0 b7b6b5b4b3b2b1b0 c7c6c5c4c3c2c1c0
转换它们到4字节里为如下所示：
0 0 a7a6a5a4a3a2 0 0 a1a0b7b6b5b4 0 0 b3b2b1b0c7c6 0 0 c5c4c3c2c1c0
然后，每个字节再加 0x20转换为可打印的字符.
eg:
从文件中读取的 3字节如下：
14 0F A8
00010100 00001111 10101000
转换为 6-bit:
000101 000000 111110 101000
每字节高两位补 0后为：
00000101 00000000 00111110 00101000
最后每字节再加 0x20，则 4字节输出应该为：
25 60 5E 48

注意： 00字节被转换为 0x60而不是 0x20.
